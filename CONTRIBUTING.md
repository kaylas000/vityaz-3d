# Contributing to VITYAZ

Thank you for considering contributing to VITYAZ: Special Operations! This guide will help you get started.

## Code of Conduct

Be respectful, inclusive, and professional. We're building a gaming community based on brotherhood and excellence.

## How to Contribute

### 1. Fork & Clone

```bash
git clone https://github.com/kaylas000/vityaz-special-operations.git
cd vityaz-special-operations
git remote add upstream https://github.com/kaylas000/vityaz-special-operations.git
```

### 2. Create Feature Branch

```bash
git checkout -b feature/your-feature-name
```

### 3. Make Changes

- Follow existing code style
- Add tests for new functionality
- Update documentation

### 4. Commit & Push

```bash
git commit -m "feat: add new feature"
git push origin feature/your-feature-name
```

### 5. Create Pull Request

- Describe your changes
- Link related issues
- Wait for code review

## Development Guidelines

### Code Style

- **Frontend:** Prettier + ESLint (React/TypeScript)
- **Backend:** ESLint (NestJS/TypeScript)
- **Smart Contracts:** Standard formatting

### Testing

```bash
# Frontend
cd frontend
npm run test

# Backend
cd backend
npm run test

# Smart Contracts
cd contracts
npm run test
```

### Commit Messages

```
feat: add new feature
fix: fix bug in component
docs: update README
style: format code
refactor: restructure module
test: add unit tests
chore: update dependencies
```

## Areas for Contribution

- **Game Mechanics:** Improve combat system, balance weapons
- **Frontend:** UI/UX improvements, new scenes
- **Backend:** API optimization, new endpoints
- **Smart Contracts:** Security audits, new features
- **Documentation:** Improve guides, add examples
- **Testing:** Increase coverage, find bugs
- **Localization:** Translate to new languages
